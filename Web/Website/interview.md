**Employer:** We rely heavily on secure API integrations and real-time data processing for our daily cash-in/cash-out operations. Walk me through how you would architect a customized, automated pipeline utilizing Python and Bash to monitor incoming network traffic for anomalies. Specifically, how would you structure the orchestration logic to parse those logs, identify threats, and trigger an immediate, tiered alert system?

**My Answer:** I would design a three-tiered pipeline. First, at the OS level, I would write a Bash script  utilizing tools like `tcpdump` or simply tailing access logs to capture incoming traffic data in real-time. This Bash script would pipe the raw data into a lightweight Python daemon. The Python script is the brain: it would parse the logs using regex and compare the request frequencies and IP behaviors against a pre-defined threshold dictionary to spot anomalies—like sudden spikes from a single origin or malformed headers. If an anomaly breaches the critical threshold, the Python script would immediately generate a structured JSON payload  detailing the timestamp, IP, and threat type. It would then use a `POST` request to push this JSON to an internal webhook—which could trigger an n8n workflow  to alert the security team via our communication channels and automatically update the firewall rules to drop packets from that IP. This keeps the solution lightweight, terminal-centric, and highly automated.
